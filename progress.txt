## Codebase Patterns
- CLI resolves `@viwo/core` types via `dist/*.d.ts` (project references with composite) — must run `bun run build` in core before CLI typecheck picks up schema changes
- `viwo.start()` orchestrates three phases: `createWorktree` → `startContainer` → `launchAgent` — each independently callable on the Viwo interface
- `initializeAgent` in agent-manager.ts returns `{ containerId, containerName }` — use this to surface container info to callers
- Phase option/result types live in `schemas.ts`: `CreateWorktreeOptions/Result`, `StartContainerOptions/Result`, `LaunchAgentOptions`
- Dockerfile is at `packages/agents/claude-code/Dockerfile`, uses Ubuntu 24.04 base image
- Bootstrap script is `packages/agents/claude-code/claude-bootstrap.sh`, copied into container as `/usr/local/bin/claude-bootstrap`
- Container runs as non-root user `claude` (UID 10001)
- Docker manager is at `packages/core/src/managers/docker-manager.ts`
- Agent manager is at `packages/core/src/managers/agent-manager.ts`
- Main SDK flow is in `packages/core/src/viwo.ts`
- Container names are deterministic: use `generateContainerName(sessionId)` from docker-manager (format: `viwo-{zero-padded sessionId}`)
- Container state dir: `{viwo-data}/container-state/{sessionId}/` — use `ensureContainerStatePath(sessionId)` from `utils/paths.ts`
- State dir is volume-bound into container at `/tmp/viwo-state`
- `createContainer` in docker-manager accepts `additionalBinds` for extra volume mounts beyond the workspace
- `startClaudeContainer` in agent-manager.ts is the single place containers are created for Claude Code
- Prompt/model are passed to containers via `VIWO_PROMPT` and `VIWO_MODEL` env vars (not CMD args)
- tmux session name inside container is `viwo` — use `tmux has-session -t viwo` to check status
- Container stays alive after Claude exits via `exec bash` fallthrough in bootstrap script
- State file inside container: `/tmp/viwo-state/viwo-state.json` — written by Claude Code hooks (PostToolUse/Notification) and by entrypoint on exit
- Claude Code hooks are configured via `~/.claude/settings.json` written by bootstrap before launch
